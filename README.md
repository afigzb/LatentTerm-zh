# LatentTerm-zh
大体是一个针对网络小说的无监督中文术语提取。目标是从百万字中文文本中，提取出非直接相关的关键词。例如输入“魂兽”得到“柔骨魅兔”

## 算法核心

LatentTerm-zh 的核心算法由四个维度的策略构成，通过多角度挖掘词汇间的潜在关联：

### 策略 A：字符包含
**逻辑**：如果一个词包含了关键词里的字符，它在某种程度上可能相关。
**代码实现**：不仅计算候选词与关键词的字符重叠比例，还会考虑**字符的相对位置匹配度**（例如在词首还是词尾）以及词频权重。重叠度越高、位置越一致，得分越高。

### 策略 B：上下文模式引导
**逻辑**：如果两个词经常出现在相同的句子结构（上下文）中，它们大概率是同类词。
**代码实现**：算法会根据种子词的左右上下文模板（如固定搭配的前后缀字词），扫描全文，寻找能填入这些高频模板的新词。同时会根据距离进行衰减计算，并结合模板的特异性进行打分。

### 策略 C：共现近邻
**逻辑**：在物理距离上经常出现在关键词附近的词，语义相关性高。
**代码实现**：统计在种子词特定窗口范围（如前后 50 个字符）内高频出现的词。为了排除常见的无意义高频词（如“的”、“了”），算法使用了 **Lift 比率**来衡量真实的关联强度。

### 策略 D：构词结构相似
**逻辑**：中文词组相同结构存在相关性。例如“强化学习”和“机器学习”共享了“学习”这个词缀。
**代码实现**：拆解关键词的核心语素，并记录它们在词中的位置（前缀、后缀、中间）。如果候选词与关键词共享相同的语素，且**语素所处的位置相同**（例如都是后缀），则会获得极高的权重加成。
